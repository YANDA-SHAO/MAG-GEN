This document records the complete process used to construct the final Kubric_vmm training dataset for the FD4MM framework. The final dataset is stored at /data/curtin_cumlg/curtin_yanda/work/datasets/Kubric_vmm. The dataset follows a unified structure consisting of five directories (frameA, frameB, frameC, amplified, and meta) together with a magnification factor file train_mf.txt. Each training sample is represented by a consistent six-digit index shared across all directories. For example, a sample with index 000123 corresponds to the files frameA/000123.png, frameB/000123.png, frameC/000123.png, amplified/000123.png, and meta/000123.json. The magnification factor for each sample is stored in train_mf.txt, where the i-th line corresponds to the sample whose index is formatted as %06d. This strict indexing scheme ensures that all input frames, ground truth frames, metadata, and magnification factors remain synchronized.

The original Kubric_vmm dataset initially contained 2075 samples with indices ranging from 000000 to 002074. Additional synthetic datasets were generated locally using Kubric and organized into several folders named Data-5, Data-6, Data-7, and Data-8. Each of these datasets originally followed a scene-based structure in which each scene directory contained a meta_scene.json file and multiple view_* subdirectories. The meta_scene.json file stored the magnification factor under the field alpha. Each view_* directory contained the images A.png, B.png, Y.png, and optionally meta_view.json. In this format, A.png and B.png represent the two input frames, while Y.png represents the magnified ground-truth frame.

Before merging with the main dataset, these scene-based datasets were converted into the standardized FD4MM dataset format using a Python conversion script. During this conversion, A.png was mapped to frameA, B.png to frameB, and Y.png to the amplified directory as the ground truth magnified frame. Because the Kubric data did not include an explicit third frame, frameC was created using Y.png as a placeholder to maintain compatibility with the existing dataset structure. Metadata files were copied or generated into the meta directory, and the magnification factor for each sample was extracted from meta_scene.json["alpha"] and written sequentially into train_mf.txt. The conversion process produced standardized datasets named vmm_train_local_data5, vmm_train_local_data6, vmm_train_local_data7, and vmm_train_local_data8.

After conversion, these datasets were packaged locally using TAR archives and uploaded to the server. Once transferred, they were extracted directly under the Kubric_vmm directory. The merging process was performed using a dedicated script (merge_into_current_kubric.py) that automatically detects the maximum existing sample index in the dataset and appends new samples sequentially. This ensures that all merged data maintains a consistent global indexing scheme without overwriting existing samples. During merging, images from the source dataset (frameA, frameB, frameC, amplified, and meta) were copied into the corresponding directories of the main dataset while assigning new indices. At the same time, the magnification factors from the source train_mf.txt file were appended to the end of the master train_mf.txt.

The datasets were merged in chronological order. The original dataset covered indices 000000–002074 (2075 samples). The dataset Data-5 added 900 samples occupying indices 002075–002974. The dataset Data-6 added 3320 samples occupying indices 002975–006294. After that, Data-7 added 1000 samples occupying indices 006295–007294, and Data-8 added 2000 samples occupying indices 007295–009294. This sequential merging process ensured that the final dataset preserved a clear mapping between index ranges and their corresponding source datasets.

After completing all merges, the final dataset integrity was verified using the script check_vmm_dataset.py. This script confirmed that the five directories (frameA, frameB, frameC, amplified, and meta) contain the same number of files and that the number of lines in train_mf.txt matches the number of samples. The final dataset contains 9294 samples in total, and the integrity check returned successful results indicating that all file index sets match and that the magnification factor file is correctly aligned with the dataset indices. Therefore, the Kubric_vmm dataset is now fully synchronized and ready for use in FD4MM model training and evaluation.
